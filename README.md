## 👋 Hey, I'm Himanshu

I'm a data and analytics engineer with **6+ years** building end-to-end pipelines and cloud warehouses on the modern stack: **Snowflake, dbt, Dagster, Python**, with a lot of Advanced SQL and dimensional modeling underneath.

What I actually love is building things the whole way through. I shipped a **production SaaS on my own**, then built a **full analytics pipeline on top of its live data**. That loop, build the product and then build the data that makes it smarter, is the work I can't put down. Because I built the product, I know what every metric is supposed to mean.

Lately I'm also working where data meets **AI/agentic engineering** (LangGraph, LLMs, RAG), always on top of real, governed data rather than toy datasets.

## 🚀 What I'm building

### [OneOnOne](https://one-on-one-alpha.vercel.app) · a B2B SaaS, built and shipped solo
`Next.js` · `Supabase` · `LLMs` · `Dodo Payments` · `Vercel`

A shared work journal that turns a year of entries into AI-generated performance reviews. I designed and built the whole thing on my own: auth, real subscription billing, AI guardrails (rate limits, cost caps, fallbacks), and a full unit + integration + E2E test suite. Live in production with real users.

### [oneonone-analytics](https://github.com/whysokara/oneonone-analytics) · the data platform behind it
`Python` · `Snowflake` · `dbt` · `MetricFlow` · `Dagster`

An end-to-end analytics pipeline on OneOnOne's live production data. Python ingestion into Snowflake, dbt transformation (17 models, 80+ tests), and a governed metrics layer, orchestrated daily in Dagster with data-quality checks surfaced as asset checks. Every bug is documented with root cause, fix, and lesson.

### Potion · end-to-end data platform for a SaaS
`dbt` · `Snowflake` · `Dagster` · `Metabase`

Turns multiple disconnected SaaS sources into a clean three-layer warehouse with board-ready KPIs (MRR, churn, CAC, LTV, retention), orchestrated in Dagster with live dashboards in Metabase.
